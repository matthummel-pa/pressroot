Test plan

A good test pass in 20 minutes.

1

Set up

Clone + build (install docs), or use the zero-install Playground: npm run wp boots WordPress at 127.0.0.1:8881.

2

Build a site

Follow a build recipe with a real or invented business. Save Theme Settings, apply a site type, AI-write all pages, generate images, hit 🎲 a few times.

3

Report

File one issue per problem with the tester label: what you did, what you expected, what you got, screenshots. Ugly counts as a bug — "deal looked bad" is valid feedback.

What we most need tested right now: the full save-to-build pipeline on fresh sites, non-English brand answers, free (Pollinations) vs. keyed providers, the restaurant/realty/affiliate types, and previews across browsers.

Open a tester application — takes two minutes. Or just install it, break it, and file issues; that counts too.
